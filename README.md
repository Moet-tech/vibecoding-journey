# vibecoding-journey

Hành trình 30 ngày tự học **vibecoding** cùng [Claude Code](https://claude.com/claude-code).

## Background

Tôi xuất thân từ ngành **kinh tế** — không phải dân kỹ thuật. Trước giờ tôi xài máy tính để đọc, viết, làm bảng tính, chứ chưa bao giờ thực sự *làm chủ* được nó. Repo này là nhật ký việc tôi đổi điều đó.

## Mục tiêu

Trong 30 ngày, mỗi ngày một folder `day-XX/`, tôi sẽ:

- Tập **Git workflow** thật sự: branch / merge / rebase / cherry-pick — không chỉ `add → commit → push`
- Cài đặt **Claude Code memory + hooks** để biến nó thành môi trường làm việc của riêng mình
- Hiểu **GitHub Actions** ở mức đủ dùng (CI cơ bản)
- Build các tool nhỏ bằng cách *vibe* với Claude Code: mô tả ý tưởng → xem code chạy → đọc → sửa → hiểu

Không phải trở thành software engineer. Mục tiêu là tự build được những thứ tôi cần cho công việc kinh tế / phân tích, và đọc hiểu được code khi cần.

## Cấu trúc

```
day-01/   ngày học đầu tiên — tools, setup, Git cơ bản
day-02/
...
day-30/
```

Mỗi folder `day-XX/` sẽ chứa: ghi chú, script nhỏ, hoặc project mini của ngày đó.

## Quy tắc với chính mình

1. Code mỗi ngày, dù chỉ 30 phút.
2. Commit mỗi ngày — push lên GitHub trước khi đi ngủ.
3. Không xóa lịch sử lỗi sai. Sai cũng commit.
4. Viết note ngắn cuối mỗi ngày: *hôm nay học được gì, vướng ở đâu*.
